# 手動テストチェックリスト

> 最終更新: 2025-01-XX  
> 目的: 全機能の動作確認チェックリスト

---

## 使い方

このチェックリストは、システムの全機能が正常に動作することを確認するためのものです。
各項目を実施し、☐を☑に変更してください。

---

## 1. 環境セットアップ

### 1.1 サービス起動確認

- [ ] SQLiteデータベースファイルが存在する（`ls ./data/mobile_app.db`）
- [ ] Web APIが起動している（Port 8080）
- [ ] Mobile BFFが起動している（Port 8081）
- [ ] Admin BFFが起動している（Port 8082）
- [ ] Admin Webが起動している（Port 3000）

### 1.2 データベース初期化

- [ ] データベースが作成されている
- [ ] 全テーブルが作成されている（6テーブル）
- [ ] 初期データが投入されている
  - [ ] ユーザー（3件以上）
  - [ ] 管理者（1件以上）
  - [ ] 商品（3件以上）
  - [ ] 機能フラグ（ユーザー数分）

## 2. 認証・認可機能

### 2.1 ユーザーログイン

- [ ] 正しいID/パスワードでログイン成功
- [ ] 誤ったパスワードでログイン失敗（401エラー）
- [ ] 存在しないユーザーIDでログイン失敗（401エラー）
- [ ] 必須項目未入力でバリデーションエラー（400エラー）

### 2.2 管理者ログイン

- [ ] 正しいID/パスワードで管理者ログイン成功
- [ ] 誤ったパスワードでログイン失敗（401エラー）
- [ ] 一般ユーザーのIDで管理者ログイン失敗

### 2.3 トークン検証

- [ ] 有効なトークンで保護されたAPIにアクセス成功
- [ ] トークンなしでAPIアクセス失敗（401エラー）
- [ ] 不正なトークンでAPIアクセス失敗（401エラー）

### 2.4 権限チェック

- [ ] 一般ユーザーが管理者APIにアクセス失敗（403エラー）
- [ ] 管理者が一般ユーザー専用APIにアクセス失敗（403エラー）
- [ ] 管理者が管理者APIにアクセス成功
- [ ] 一般ユーザーが一般ユーザーAPIにアクセス成功

## 4. 商品購入機能

### 4.1 基本的な購入フロー

- [ ] 商品詳細から購入ボタンをタップ
- [ ] 購入数量を入力できる（100個単位）
- [ ] 合計金額が表示される
- [ ] 購入確定ボタンをタップ
- [ ] 購入完了メッセージが表示される
- [ ] 購入データがDBに保存される

### 4.2 購入数量バリデーション

- [ ] 100個単位以外の数量で購入失敗（400エラー）
  - [ ] 99個
  - [ ] 101個
  - [ ] 150個
- [ ] 範囲外の数量で購入失敗（400エラー）
  - [ ] 99個以下
  - [ ] 10000個以上
- [ ] マイナス値で購入失敗（400エラー）
- [ ] 0個で購入失敗（400エラー）

### 4.3 購入数量範囲

- [ ] 最小値（100個）で購入成功
- [ ] 最大値（9900個）で購入成功
- [ ] 中間値（500個、1000個など）で購入成功

### 4.4 合計金額計算

- [ ] 合計金額が正しく計算される
  - [ ] 単価 × 数量 = 合計金額
- [ ] 購入時の単価がDBに保存される
- [ ] 商品の単価変更後も過去の購入データは変わらない

### 4.5 購入履歴

- [ ] 購入後、購入履歴に表示される
- [ ] 購入日時が記録される
- [ ] 複数回購入した場合、すべて表示される
- [ ] 商品名、数量、単価、合計金額が表示される
