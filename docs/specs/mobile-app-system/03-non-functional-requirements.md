## 1. 非機能要件概要

本ドキュメントでは、mobile-app-systemの非機能要件を定義します。
デモンストレーション用途であることを考慮し、実現可能な範囲で品質基準を設定します。

## 2. パフォーマンス要件

### 2.1 応答時間

**NFR-001: API応答時間**
- **要件**: 各API呼び出しの応答時間基準
- **優先度**: 高

| API種別 | 目標応答時間 | 最大許容時間 |
|---------|-------------|-------------|
| ログイン | 1秒以内 | 2秒 |
| 商品一覧取得 | 2秒以内 | 3秒 |
| 商品検索 | 1秒以内 | 2秒 |
| 商品詳細取得 | 1秒以内 | 2秒 |
| 商品購入 | 3秒以内 | 5秒 |
| お気に入り登録/解除 | 1秒以内 | 2秒 |
| 商品更新（管理） | 1秒以内 | 2秒 |
| 機能フラグ変更 | 0.5秒以内 | 1秒 |

**測定条件**:
- ネットワーク: ローカルネットワーク（LAN）
- 負荷: 同時接続10ユーザー
- データ件数: 商品100件、ユーザー100件

**測定方法**:
- サーバー側ログで処理時間を記録
- クライアント側でレスポンス受信までの時間を記録

---

**NFR-002: 画面表示時間**
- **要件**: ユーザーが画面遷移時に待つ最大時間
- **優先度**: 中

| 画面 | 目標表示時間 | 最大許容時間 |
|------|-------------|-------------|
| ログイン画面 | 即座 | 1秒 |
| 商品一覧画面 | 2秒以内 | 3秒 |
| 商品詳細画面 | 1秒以内 | 2秒 |
| 購入画面 | 即座 | 1秒 |
| 管理画面 | 2秒以内 | 3秒 |

**UX考慮**:
- 1秒以上かかる場合、ローディングインジケータを表示
- 3秒以上かかる場合、タイムアウトエラーを表示

### 2.2 スループット

**NFR-010: 同時接続ユーザー数**
- **要件**: システムが同時に処理できるユーザー数
- **優先度**: 低（デモ用途）

| 指標 | 目標値 | 説明 |
|------|-------|------|
| 同時接続ユーザー数 | 100ユーザー | 同時にログイン状態のユーザー |
| 同時APIリクエスト | 50リクエスト/秒 | 瞬間的な同時リクエスト |
| DB接続プールサイズ | 20接続 | 同時DB接続数 |

**デモ用途のため、以下は考慮しない**:
- 大規模スケーリング
- ピーク時の負荷対策
- オートスケーリング

- **優先度**: 低

| データ種別 | 想定件数 | 最大件数 |
|-----------|---------|---------|
| 商品 | 100件 | 1,000件 |
| ユーザー | 100件 | 1,000件 |
| 購入履歴 | 10,000件 | 100,000件 |
| お気に入り | 1,000件 | 10,000件 |

## 5. セキュリティ要件

### 5.1 認証・認可

**NFR-050: 認証方式**
- **要件**: JWT認証の実装
- **優先度**: 高

| 項目 | 仕様 |
|------|------|
| 認証方式 | JWT (JSON Web Token) |
| トークン有効期限 | 24時間 |
| トークン更新 | なし（再ログイン必要） |
| 署名アルゴリズム | HS256 |
| シークレットキー | 環境変数で管理 |

詳細は `08-security.md` を参照

---

**NFR-051: パスワードポリシー**
- **要件**: パスワードの要件
- **優先度**: 中

| 項目 | 要件 |
|------|------|
| 最小文字数 | 8文字 |
| 最大文字数 | 50文字 |
| 複雑性要件 | なし（デモ用途） |
| パスワード変更 | 未実装 |
| パスワードハッシュ | bcrypt |
| ソルト | bcryptが自動生成 |

---

**NFR-052: 権限管理**
- **要件**: ユーザー種別による権限制御
- **優先度**: 高

| ユーザー種別 | 権限 |
|------------|------|
| エンドユーザー (user) | モバイルアプリ機能のみ |
| 管理者 (admin) | 管理機能のみ |

権限マトリクスの詳細は `02-functional-requirements.md` の FR-201 を参照

---

### 5.2 データ保護

**NFR-060: データ暗号化**
- **要件**: データの暗号化
- **優先度**: 中

| データ種別 | 暗号化 | 方式 |
|-----------|-------|------|
| 通信（Transport） | TLS 1.2以上 | HTTPS |
| パスワード | ハッシュ化 | bcrypt |
| JWTトークン（モバイル） | 暗号化ストレージ | Keychain/EncryptedSharedPreferences |
| JWTトークン（Web） | なし | localStorage（デモ用途） |
| DB内データ | なし | 平文（デモ用途） |

**デモ用途のため、以下は実装しない**:
- DB暗号化（Transparent Data Encryption）
- 個人情報のマスキング

---

**NFR-061: 入力バリデーション**
- **要件**: 全ての入力に対するバリデーション
- **優先度**: 高

| バリデーション種別 | 実装箇所 |
|------------------|---------|
| クライアント側 | フロントエンド（モバイル/Web） |
| サーバー側 | Web API（必須） |

**バリデーション項目**:
- 必須チェック
- 型チェック
- 文字数制限
- 形式チェック
- 範囲チェック
- SQLインジェクション対策
- XSS対策

詳細は `08-security.md` を参照

---

### 5.3 セキュリティ対策

**NFR-070: OWASP Top 10 対策**
- **要件**: OWASP Top 10 の主要な脆弱性への対策
- **優先度**: 高

| 脆弱性 | 対策 | 実装状況 |
|-------|------|---------|
| A01: アクセス制御の不備 | JWT認証、権限制御 | 実装 |
| A02: 暗号化の失敗 | HTTPS、bcrypt | 実装 |
| A03: インジェクション | パラメータ化クエリ、バリデーション | 実装 |
| A04: 安全でない設計 | セキュアな設計原則 | 考慮 |
| A05: セキュリティ設定ミス | デフォルト設定の変更 | 実装 |
| A06: 脆弱性のある古いコンポーネント | 最新ライブラリ使用 | 実装 |
| A07: 識別と認証の失敗 | JWT、セッション管理 | 実装 |
| A08: ソフトウェアとデータの整合性の不備 | 入力バリデーション | 実装 |
| A09: セキュリティログとモニタリングの失敗 | ログ出力 | 最小限実装 |
| A10: サーバサイドリクエストフォージェリ | 未対応（該当なし） | - |

詳細は `08-security.md` を参照

---

**NFR-071: セッション管理**
- **要件**: セッションの安全な管理
- **優先度**: 高

| 項目 | 仕様 |
|------|------|
| セッションID | JWTトークン |
| 有効期限 | 24時間 |
| 自動延長 | なし |
| ログアウト時 | トークン削除 |
| タイムアウト | 24時間後に自動ログアウト |
| 複数デバイスログイン | 許可（デモ用途） |

## 6. 保守性要件

### 6.1 ログ出力

**NFR-080: ログ出力**
- **要件**: 適切なログ出力
- **優先度**: 高

**ログレベル**:
| レベル | 用途 | 出力先 |
|-------|------|-------|
| ERROR | エラー情報 | ファイル、コンソール |
| WARN | 警告情報 | ファイル、コンソール |
| INFO | 一般情報 | ファイル |
| DEBUG | デバッグ情報 | ファイル（開発時のみ） |

**ログ出力項目**:
- タイムスタンプ
- ログレベル
- コンポーネント名
- メッセージ
- ユーザーID（該当する場合）
- リクエストID（該当する場合）
- スタックトレース（エラー時）

**ログローテーション**:
- ファイルサイズ: 100MB
- 保持世代: 7世代
- 保持期間: 30日

---

**NFR-081: ログ内容**
- **要件**: 各種イベントのログ記録
- **優先度**: 中

| イベント | ログレベル | 記録内容 |
|---------|----------|---------|
| ログイン成功 | INFO | ユーザーID、タイムスタンプ |
| ログイン失敗 | WARN | ユーザーID、理由 |
| API呼び出し | DEBUG | エンドポイント、パラメータ |
| API エラー | ERROR | エンドポイント、エラー内容、スタックトレース |
| DB エラー | ERROR | クエリ、エラー内容 |
| 購入処理 | INFO | ユーザーID、商品ID、個数、金額 |
| 商品更新 | INFO | 管理者ID、商品ID、変更内容 |
| 機能フラグ変更 | INFO | 管理者ID、ユーザーID、フラグ内容 |
