## 1. 用語集について

本ドキュメントでは、mobile-app-systemで使用される専門用語、略語、概念を定義します。
プロジェクト関係者間で用語の統一を図り、認識の齟齬を防ぐことを目的とします。

---

## 2. ビジネス用語

### エンドユーザー（End User）
- **英語**: End User
- **略語**: なし
- **定義**: モバイルアプリを使用して商品を閲覧・購入する一般ユーザー
- **関連用語**: ユーザー、モバイルユーザー
- **備考**: 「user」権限を持つユーザー

---

### 管理者（Administrator）
- **英語**: Administrator
- **略語**: Admin
- **定義**: 管理Webアプリを使用して商品管理や機能フラグ管理を行うシステム管理者
- **関連用語**: アドミン、管理ユーザー
- **備考**: 「admin」権限を持つユーザー

---

### 商品（Product）
- **英語**: Product
- **略語**: なし
- **定義**: アプリ上で販売される商品。商品名、単価、説明文、画像URLを持つ
- **関連用語**: 商品マスタ
- **備考**: 商品の追加・削除は本システムのスコープ外

---

### 単価（Unit Price）
- **英語**: Unit Price
- **略語**: なし
- **定義**: 商品1個あたりの価格（円）
- **関連用語**: 価格、値段
- **備考**: 整数値、1円以上

---

### 購入（Purchase）
- **英語**: Purchase
- **略語**: なし
- **定義**: ユーザーが商品を購入する行為。100個単位でのみ購入可能
- **関連用語**: 注文、購入履歴
- **備考**: 実際の決済処理は行わない（デモ用途）

---

### 購入個数（Quantity）
- **英語**: Quantity
- **略語**: Qty
- **定義**: 1回の購入で購入する商品の個数。100の倍数のみ
- **関連用語**: 数量
- **備考**: 最小100個、最大9900個

---

### 合計金額（Total Amount）
- **英語**: Total Amount
- **略語**: なし
- **定義**: 単価 × 購入個数 で計算される購入時の合計金額
- **関連用語**: 総額
- **備考**: 整数値（円）

---

### お気に入り（Favorite）
- **英語**: Favorite
- **略語**: Fav
- **定義**: ユーザーが気に入った商品を保存する機能
- **関連用語**: ブックマーク、ウィッシュリスト
- **備考**: 機能フラグで制御される機能

---

### 機能フラグ（Feature Flag）
- **英語**: Feature Flag
- **略語**: FF
- **定義**: 特定の機能をユーザー単位でON/OFF制御する仕組み
- **関連用語**: フィーチャートグル、機能スイッチ
- **備考**: 段階的リリース、A/Bテストに使用

---

## 3. 技術用語

### BFF（Backend For Frontend）
- **英語**: Backend For Frontend
- **略語**: BFF
- **定義**: フロントエンド（モバイルアプリ、Webアプリ）専用のバックエンドAPI層
- **関連用語**: APIゲートウェイ
- **備考**: 本システムではMobile BFFとAdmin BFFが存在

---

### JWT（JSON Web Token）
- **英語**: JSON Web Token
- **略語**: JWT
- **定義**: JSON形式のトークンを用いた認証方式。署名により改ざん検知可能
- **関連用語**: トークン認証、Bearer Token
- **備考**: 本システムではHS256アルゴリズムを使用

---

### トークン（Token）
- **英語**: Token
- **略語**: なし
- **定義**: 認証・認可に使用される文字列。本システムではJWTトークン
- **関連用語**: アクセストークン
- **備考**: Authorizationヘッダーで送信（Bearer方式）

---

### API（Application Programming Interface）
- **英語**: Application Programming Interface
- **略語**: API
- **定義**: アプリケーション間でデータをやり取りするためのインターフェース
- **関連用語**: REST API、Web API
- **備考**: 本システムではREST形式のHTTP APIを使用

---

### REST（REpresentational State Transfer）
- **英語**: REpresentational State Transfer
- **略語**: REST
- **定義**: HTTPメソッド（GET, POST, PUT, DELETE）を使ったWebサービスのアーキテクチャスタイル
- **関連用語**: RESTful API
- **備考**: リソース指向の設計

---

### エンドポイント（Endpoint）
- **英語**: Endpoint
- **略語**: なし
- **定義**: APIへアクセスするためのURL
- **関連用語**: API URL
- **備考**: 例: `/api/v1/products`

---

### HTTPメソッド（HTTP Method）
- **英語**: HTTP Method
- **略語**: なし
- **定義**: HTTPリクエストの種類。GET（取得）、POST（作成）、PUT（更新）、DELETE（削除）
- **関連用語**: HTTPバーブ
- **備考**: RESTfulなAPI設計で使用

---

### ペイロード（Payload）
- **英語**: Payload
- **略語**: なし
- **定義**: APIリクエスト/レスポンスの本体データ。JSONフォーマット
- **関連用語**: リクエストボディ、レスポンスボディ
- **備考**: 通常JSON形式

---

### バリデーション（Validation）
- **英語**: Validation
- **略語**: なし
- **定義**: 入力データが正しい形式・範囲であるかを検証する処理
- **関連用語**: 入力検証、妥当性確認
- **備考**: クライアント側とサーバー側の両方で実施

---

### 認証（Authentication）
- **英語**: Authentication
- **略語**: AuthN
- **定義**: ユーザーが本人であることを確認するプロセス。ログインによる本人確認
- **関連用語**: ログイン、本人確認
- **備考**: 本システムではID/パスワード + JWT

---

### 認可（Authorization）
- **英語**: Authorization
- **略語**: AuthZ
- **定義**: 認証されたユーザーが特定のリソースにアクセスする権限があるかを確認するプロセス
- **関連用語**: アクセス制御、権限制御
- **備考**: 本システムではユーザー種別（user/admin）で制御

---

### ロールベースアクセス制御（Role-Based Access Control）
- **英語**: Role-Based Access Control
- **略語**: RBAC
- **定義**: ユーザーのロール（役割）に基づいてアクセス権限を制御する方式
- **関連用語**: 権限管理
- **備考**: 本システムではuser/adminの2ロール

---

### セッション（Session）
- **英語**: Session
- **略語**: なし
- **定義**: ユーザーのログイン状態を維持する仕組み
- **関連用語**: ログイン状態
- **備考**: 本システムではJWTトークンでセッション管理

---

### ORM（Object-Relational Mapping）
- **英語**: Object-Relational Mapping
- **略語**: ORM
- **定義**: オブジェクト指向プログラミング言語とリレーショナルデータベースの間でデータを変換する技術
- **関連用語**: JPA、Hibernate
- **備考**: 本システムではSpring Data JPAを使用

---

### トランザクション（Transaction）
- **英語**: Transaction
- **略語**: Tx
- **定義**: 一連のデータベース操作をひとまとまりとして扱う単位。全成功または全失敗（ロールバック）
- **関連用語**: ACID特性
- **備考**: データ整合性を保証

---

### インデックス（Index）
- **英語**: Index
- **略語**: なし
- **定義**: データベースの検索速度を向上させるためのデータ構造
- **関連用語**: データベースインデックス
- **備考**: 主キー、外部キー、検索頻度の高い列に設定

---

### 外部キー（Foreign Key）
- **英語**: Foreign Key
- **略語**: FK
- **定義**: 他のテーブルの主キーを参照する列。テーブル間のリレーションシップを定義
- **関連用語**: 参照整合性
- **備考**: ON DELETE CASCADE/RESTRICTで削除時の動作を制御

---

### マイグレーション（Migration）
- **英語**: Migration
- **略語**: なし
- **定義**: データベーススキーマのバージョン管理と変更適用の仕組み
- **関連用語**: スキーマ変更、DBマイグレーション
- **備考**: 本システムでは初期化スクリプトのみ（マイグレーションツール不使用）

---

### レスポンシブデザイン（Responsive Design）
- **英語**: Responsive Design
- **略語**: RWD
- **定義**: 画面サイズに応じてレイアウトが最適化されるデザイン手法
- **関連用語**: 適応的デザイン
- **備考**: 管理Webアプリで部分的に適用

---

## 4. アーキテクチャ用語

### マイクロサービス（Microservices）
- **英語**: Microservices
- **略語**: なし
- **定義**: アプリケーションを小さな独立したサービスに分割するアーキテクチャパターン
- **関連用語**: サービス指向アーキテクチャ
- **備考**: 本システムはBFFパターンを採用

---

### レイヤードアーキテクチャ（Layered Architecture）
- **英語**: Layered Architecture
- **略語**: なし
- **定義**: アプリケーションを複数の層（プレゼンテーション層、ビジネスロジック層、データアクセス層等）に分割するアーキテクチャパターン
- **関連用語**: 階層化アーキテクチャ、3層アーキテクチャ
- **備考**: 本システムの基本アーキテクチャ

---

### 依存性注入（Dependency Injection）
- **英語**: Dependency Injection
- **略語**: DI
- **定義**: オブジェクトの依存関係を外部から注入する設計パターン
- **関連用語**: IoC（制御の反転）
- **備考**: Spring Bootで標準サポート

---

### DTO（Data Transfer Object）
- **英語**: Data Transfer Object
- **略語**: DTO
- **定義**: レイヤー間でデータを転送するためのオブジェクト
- **関連用語**: ValueObject
- **備考**: APIリクエスト/レスポンスで使用

---

## 5. セキュリティ用語

### ハッシュ化（Hashing）
- **英語**: Hashing
- **略語**: なし
- **定義**: データを一方向の不可逆な変換により固定長の文字列に変換すること
- **関連用語**: ハッシュ関数
- **備考**: パスワードの保存に使用（bcrypt）

---

### ソルト（Salt）
- **英語**: Salt
- **略語**: なし
- **定義**: ハッシュ化の際に追加するランダムな文字列。レインボーテーブル攻撃を防ぐ
- **関連用語**: ランダムソルト
- **備考**: bcryptが自動生成

---

### XSS（Cross-Site Scripting）
- **英語**: Cross-Site Scripting
- **略語**: XSS
- **定義**: Webアプリケーションの脆弱性を悪用し、悪意のあるスクリプトを埋め込む攻撃
- **関連用語**: クロスサイトスクリプティング
- **備考**: 出力エスケープで対策

---

### SQLインジェクション（SQL Injection）
- **英語**: SQL Injection
- **略語**: SQLi
- **定義**: SQLクエリに悪意のあるSQL文を挿入する攻撃
- **関連用語**: インジェクション攻撃
- **備考**: パラメータ化クエリで対策

---

### CSRF（Cross-Site Request Forgery）
- **英語**: Cross-Site Request Forgery
- **略語**: CSRF、XSRF
- **定義**: ユーザーの意図しないリクエストを攻撃者が送信させる攻撃
- **関連用語**: クロスサイトリクエストフォージェリ
- **備考**: JWT（Authorizationヘッダー）使用により影響軽減

---

### HTTPS（HTTP Secure）
- **英語**: HTTP Secure
- **略語**: HTTPS
- **定義**: SSL/TLSで暗号化されたHTTP通信
- **関連用語**: TLS、SSL
- **備考**: 本番環境で必須

---

### TLS（Transport Layer Security）
- **英語**: Transport Layer Security
- **略語**: TLS
- **定義**: 通信を暗号化するプロトコル
- **関連用語**: SSL（旧名称）
- **備考**: TLS 1.2以上を使用

---
