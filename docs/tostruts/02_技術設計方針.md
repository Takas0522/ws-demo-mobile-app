## 3. レイヤー設計

### 3.1 DAO 層（JDBC 直接アクセス）

```java
// DatabaseUtil.java - コネクション管理
public class DatabaseUtil {
    private static String dbPath;

    public static void initialize(String path) {
        dbPath = path;
    }

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection("jdbc:sqlite:" + dbPath);
    }
}

// BaseDao.java - 共通 DAO
public abstract class BaseDao {
    protected QueryRunner queryRunner = new QueryRunner();

    protected Connection getConnection() throws SQLException {
        return DatabaseUtil.getConnection();
    }
}

// ProductDao.java - 例
public class ProductDao extends BaseDao {
    public List<Product> findAll(int offset, int limit) throws SQLException {
        Connection conn = null;
        try {
            conn = getConnection();
            return queryRunner.query(conn,
                "SELECT * FROM products ORDER BY product_id LIMIT ? OFFSET ?",
                new BeanListHandler<Product>(Product.class),
                limit, offset);
        } finally {
            DbUtils.closeQuietly(conn);
        }
    }
}
```

### 3.2 Service 層

```java
// ProductService.java - 例
public class ProductService {
    private ProductDao productDao = new ProductDao();
    private ProductPriceHistoryDao priceHistoryDao = new ProductPriceHistoryDao();

    public List<Product> getProducts(int page, int limit) {
        try {
            int offset = (page - 1) * limit;
            return productDao.findAll(offset, limit);
        } catch (SQLException e) {
            throw new RuntimeException("商品一覧の取得に失敗しました", e);
        }
    }
}
```

### 3.3 Action 層（ViewModel）

```java
// ProductListAction.java - 管理画面用 Action
public class ProductListAction extends ActionSupport {
    private ProductService productService = new ProductService();

    // Input (リクエストパラメータ)
    private int page = 1;
    private int pageSize = 20;

    // Output (View 用データ)
    private List<Product> products;
    private int totalCount;
    private int totalPages;

    public String execute() {
        products = productService.getProducts(page, pageSize);
        totalCount = productService.getProductCount();
        totalPages = (totalCount + pageSize - 1) / pageSize;
        return SUCCESS;
    }

    // getter / setter 省略
}
```

## 4. 認証設計

### 4.1 管理画面（セッション認証）

```java
// AuthInterceptor.java - ログインチェック
public class AuthInterceptor extends AbstractInterceptor {
    @Override
    public String intercept(ActionInvocation invocation) throws Exception {
        Map<String, Object> session = invocation.getInvocationContext().getSession();
        Object user = session.get("loginUser");

        if (user == null) {
            return "login";
        }

        return invocation.invoke();
    }
}
```

### 4.2 Web API（JWT 認証）

```java
// JwtInterceptor.java - JWT 検証
public class JwtInterceptor extends AbstractInterceptor {
    @Override
    public String intercept(ActionInvocation invocation) throws Exception {
        HttpServletRequest request = ServletActionContext.getRequest();
        String authHeader = request.getHeader("Authorization");

        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            return "unauthorized";
        }

        String token = authHeader.substring(7);
        try {
            Claims claims = JwtUtil.parseToken(token);
            // リクエストにユーザー情報を格納
            request.setAttribute("userId", claims.get("userId"));
            request.setAttribute("userType", claims.get("userType"));
        } catch (Exception e) {
            return "unauthorized";
        }

        return invocation.invoke();
    }
}
```

## 5. 既存 Web API との共存

Struts2 アプリケーションは **ポート 8082** で動作し、既存の Web API（ポート 8080）とは独立して稼働する。

```
モバイルアプリ (iOS / Android / Windows)
    ↓
Mobile BFF (Spring Boot, port 8081)
    ↓
Web API (Spring Boot, port 8080) ← 変更なし
    ↓
SQLite Database

管理者ブラウザ
    ↓
Struts2 アプリ (Tomcat, port 8082) ← 新規
    ↓
SQLite Database（直接アクセス）

（内部システム）
    ↓
Struts2 Web API (同一 WAR, port 8082) ← 新規
    ↓
SQLite Database（直接アクセス）
```

> **注意**: SQLite はファイルベースの DB であり、同時書き込みに対するロック制御が必要。Struts2 アプリと既存 Web API が同一 DB ファイルに同時アクセスする場合、WAL モードの有効化や適切なコネクション管理が必要となる。
