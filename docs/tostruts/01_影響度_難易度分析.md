# 影響度・難易度分析

## 1. コンポーネント別影響度

### 1.1 admin-web（Vue 3 + TypeScript）— 影響度：**極大**

| 項目 | 詳細 |
|---|---|
| 対応 | **完全廃止 → JSP で再実装** |
| 影響範囲 | 6 画面（Login, Dashboard, ProductList, ProductEdit, UserList, FeatureFlag）、5 コンポーネント、API クライアント、Pinia ストア、ルーティング |
| 作業内容 | Vue SFC → JSP への全面書き換え。リアクティブ UI → サーバーサイドレンダリングへの移行。Axios → Struts2 Action 呼び出しへの変更 |
| リスク | UI/UX の品質低下（SPA → MPA 遷移）、クライアントサイドバリデーションの再実装 |

#### 画面別移行マッピング

| Vue View | 移行先 JSP | 機能 | 複雑度 |
|---|---|---|---|
| LoginView.vue | login.jsp | ログインフォーム（バリデーション付き） | 低 |
| DashboardView.vue | dashboard.jsp | メニューカード表示 | 低 |
| ProductListView.vue | product-list.jsp | 商品一覧＋ページネーション | 中 |
| ProductEditView.vue | product-edit.jsp | 商品編集フォーム＋バリデーション | 中 |
| UserListView.vue | user-list.jsp | ユーザー一覧＋ページネーション | 中 |
| FeatureFlagView.vue | feature-flag.jsp | フラグ ON/OFF トグル＋通知 | 高 |

## 2. 難易度分析

### 2.1 全体難易度：**極めて高い**

| 難易度要因 | 評価 | 説明 |
|---|---|---|
| JDK 1.7 ダウングレード | ★★★★★ | 現行 Java 17 → 1.7 は 6 世代のダウングレード。利用可能なライブラリが大幅に制限される |
| フレームワーク変更 | ★★★★☆ | Spring Boot → Struts2 は根本的なアーキテクチャ変更。DI コンテナ、ORM、セキュリティすべてが変わる |
| フロントエンド移行 | ★★★★☆ | Vue 3 SPA → JSP MPA は UI パラダイムの完全な変更。リアクティブ → リクエスト/レスポンス |
| DB アクセス層再構築 | ★★★☆☆ | JPA → JDBC 直接。SQL を手書きする必要がある |
| 認証方式変更 | ★★★☆☆ | JWT → セッション（管理画面）。Struts2 API は JWT を Java 7 で再実装 |
| テスト環境構築 | ★★★☆☆ | JUnit 4 ベースへの移行。モダンなテスト支援ツールが使えない |

### 2.2 技術的リスク詳細

#### JDK 1.7 互換性リスク

| ライブラリ | Java 7 互換バージョン | 備考 |
|---|---|---|
| Struts2 | 2.3.37 (最終 2.3 系) | **セキュリティパッチが終了** — CVE 対応なし |
| SQLite JDBC | 3.8.11.2 | Java 7 互換の最終バージョン付近 |
| Jackson | 2.4.x | Java 7 対応の最終メジャーバージョン |
| bcrypt (jBCrypt) | 0.4 | Java 7 互換 |
| JJWT | 0.6.0 以下 | Java 7 対応バージョンは機能が限定的 |
| Servlet API | 3.0 / 3.1 | Tomcat 7 対応 |
| JSTL | 1.2 | Java 7 互換 |
| Commons DbUtils | 1.6 | Java 7 互換 |
| Log4j | 1.2.x or SLF4J 1.7.x | Java 7 互換 |

> **⚠️ 重大な懸念**: Struts2 2.3.x 系はセキュリティサポートが終了しており、既知の脆弱性（OGNL インジェクション等）に対するパッチが提供されない。本番環境への適用にはセキュリティリスクが伴う。

#### OGNL インジェクション脆弱性

Struts2 2.3.x 系には複数の重大な OGNL インジェクション脆弱性が報告されている：
- CVE-2017-5638（S2-045）: Remote Code Execution
- CVE-2017-9805（S2-052）: Remote Code Execution
- CVE-2018-11776（S2-057）: Remote Code Execution

これらは 2.3.x の最終パッチで一部対応されているが、新たな脆弱性が発見された場合のパッチ提供は期待できない。

## 3. 並行実装可能性分析

### 3.1 依存関係マップ

```
[Phase 0] プロジェクト基盤構築
    ├── Maven プロジェクト作成
    ├── Struts2 設定
    └── DB 接続設定
         │
    ┌────┴────┐
    │         │
[Phase 1A] [Phase 1B]
DAO 層     共通部品
構築       (認証, 例外処理)
    │         │
    └────┬────┘
         │
    ┌────┼─────────┐
    │    │         │
[Phase 2A] [Phase 2B] [Phase 2C]
管理画面   管理画面    Struts2
Action +   JSP View   Web API
Service    構築       (JSON)
    │    │         │
    └────┼─────────┘
         │
[Phase 3] 結合テスト・移行
```

### 3.2 並行作業の可否

| 作業ペア | 並行可否 | 条件 |
|---|---|---|
| DAO 層構築 ↔ 共通部品（認証等） | ✅ 可能 | 相互に独立 |
| 管理画面 Action ↔ JSP View | ⚠️ 部分的に可能 | Action インターフェース（getter/setter）を先に定義すれば View は並行開発可能 |
| 管理画面 Action ↔ Struts2 Web API | ✅ 可能 | Service/DAO 層を共有するが Action は独立 |
| 管理画面全体 ↔ Struts2 Web API | ✅ 可能 | Service/DAO 層完成後は完全並行可能 |
| JSP View 間（各画面） | ✅ 可能 | 各画面は独立 |
| Service 層（各ドメイン） | ✅ 可能 | ドメイン単位で独立開発可能 |

### 3.3 推奨チーム構成

| チーム | 担当 | 人数目安 |
|---|---|---|
| チーム A | 基盤 + DAO + 共通部品 → 管理画面 Action/Service | 2 名 |
| チーム B | JSP View + CSS + クライアントサイド | 1-2 名 |

## 4. 移行による喪失機能・品質低下

| 項目 | 現行 | 移行後 | 影響 |
|---|---|---|---|
| SPA UX | ページ遷移なしの滑らかな操作 | MPA（画面遷移ごとにリロード） | UX 低下 |
| リアクティブ UI | Vue 3 のリアクティブ更新 | フォーム POST → リダイレクト | 操作性低下 |
| 型安全性 | TypeScript による型チェック | JSP は型安全性なし | 品質リスク |
| クライアントバリデーション | 即座のフィードバック | サーバーサイドバリデーション中心 | UX 低下 |
| ホットリロード | Vite による高速 HMR | JSP 変更時は再デプロイ | 開発効率低下 |
| セキュリティ | Spring Security + 最新パッチ | Struts2 2.3.x（パッチ終了） | **セキュリティリスク** |
| エコシステム | Spring Boot の豊富なスターター | 限定的なプラグイン | 拡張性低下 |
| Java 言語機能 | ラムダ・Streams・var 等 | try-with-resources まで | 生産性低下 |
