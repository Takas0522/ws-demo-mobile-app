# Struts2 移行プロジェクト概要

## 1. 目的

現行の管理者向け Web アプリケーション構成（Vue.js + Admin BFF + Web API の三層構造）を、Apache Struts2 ベースのモノリシックアプリケーションに再構成する。

## 2. 移行の全体像

### 現行アーキテクチャ（As-Is）

```
管理者 Web アプリ (Vue 3 + TypeScript, port 5173)
    ↓ Axios / REST
Admin BFF (Spring Boot 3.2.1, Java 17, port 8082)
    ↓ WebClient / REST
Web API (Spring Boot 3.2.1, Java 17, port 8080)
    ↓ JPA
SQLite Database
```

### 移行後アーキテクチャ（To-Be）

```
管理者 Web アプリ (Struts2 JSP, port 8082)
    ↓ Action / Service / DAO（直接 DB アクセス）
SQLite Database

Struts2 Web API (JSON レスポンス, 同一 WAR 内)
    ↓ Service / DAO（直接 DB アクセス）
SQLite Database

※ 既存の Web API (Spring Boot, port 8080) は存続
※ Mobile BFF / モバイルアプリ / Windows アプリへの影響なし
```

## 3. 移行対象と非対象

| コンポーネント | 対応 | 説明 |
|---|---|---|
| admin-web (Vue 3) | **廃止 → JSP へ移行** | Struts2 の View (JSP) として再実装 |
| admin-bff (Spring Boot) | **廃止** | Struts2 アプリが DB を直接参照する構成に変更 |
| Struts2 管理アプリ | **新規作成** | admin-web + admin-bff を統合した Struts2 アプリ |
| Struts2 Web API | **新規作成** | 内部向け REST API を Struts2 で構築（同一 WAR）|
| web-api (Spring Boot) | **変更なし** | 既存のまま存続（モバイル / Windows 向け） |
| mobile-bff (Spring Boot) | **変更なし** | 既存のまま存続 |
| Android / iOS アプリ | **変更なし** | mobile-bff 経由のため影響なし |
| Windows アプリ | **変更なし** | CppApiServer / mobile-bff 経由のため影響なし |
| SQLite データベース | **変更なし** | スキーマ・データ共に変更なし |

## 4. 技術スタック変更

| 項目 | 現行 | 移行後 |
|---|---|---|
| Java バージョン | JDK 17 | **JDK 1.7** |
| フレームワーク（管理画面） | Spring Boot 3.2.1 + Vue 3 | **Apache Struts 2.3.37** |
| View 技術 | Vue 3 SFC + TypeScript | **JSP + JSTL + Struts2 タグ** |
| OR マッピング | Spring Data JPA + Hibernate 6 | **JDBC 直接 / Apache Commons DbUtils** |
| 認証方式 | JWT (stateless) | **HttpSession (stateful)** |
| ビルドツール | Maven + npm/Vite | **Maven + WAR パッケージ** |
| アプリケーションサーバー | 組込み Tomcat (Spring Boot) | **Apache Tomcat 7.x (外部)** |
| JSON 処理 | Jackson (Spring 組込み) | **Jackson 2.4.x / Struts2 JSON Plugin** |
| SQLite ドライバ | sqlite-jdbc 3.45.1.0 | **sqlite-jdbc 3.8.x 系（Java 7 互換）** |

## 5. JDK 1.7 制約に伴う注意事項

JDK 1.7 への変更により、以下の制約が発生する：

- **Spring Boot は使用不可**（最低 Java 8 が必要）
- **Lombok は使用不可**（Java 8 以降が必要）
- **Lambda 式・Stream API は使用不可**（Java 8 以降の機能）
- **try-with-resources** は使用可能（Java 7 で導入）
- **Diamond 演算子 `<>`** は使用可能（Java 7 で導入）
- Struts2 は **2.3.x 系**が Java 7 の最終対応バージョン（2.5 以降は Java 8 必須）
- Hibernate 5 以降は Java 8 必須のため、**Hibernate 4.x** または **JDBC 直接**を使用
- 使用可能なライブラリバージョンが全般的に古くなる

## 6. MVVM パターンの解釈（Struts2 コンテキスト）

本プロジェクトでは、Struts2 における MVVM パターンを以下のように解釈する：

| MVVM レイヤー | Struts2 における実装 |
|---|---|
| **Model** | Entity / DTO クラス、Service 層、DAO 層 |
| **View** | JSP + JSTL + Struts2 タグライブラリ |
| **ViewModel** | Struts2 Action クラス（フォームバインディング + View 用データ準備） |

Struts2 の Action クラスは、フォームデータのバインディングと View に渡すデータの準備を担う ViewModel として機能する。ビジネスロジックは Service 層に委譲し、Action は薄いレイヤーとする。
