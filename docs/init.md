# アプリケーション仕様について

ネイティブモバイルアプリとそのアプリの挙動と商品を管理する管理者用Webアプリケーションからなるアプリケーションです。

## アプリケーションの仕様について

- ユーザーはログインを行うことで商品一覧画面にすすむことができます
- ユーザーは商品一覧画面で検索を行い商品を探すことが可能です
- ユーザーは商品一覧画面から商品の詳細画面に移動することができ、そこで商品を購入することが可能です
- 商品には単価が設定されており単価に個数をかけることで商品を購入することが可能です
    - 商品の個数は1個単位ではなく100個単位でのみ購入することが可能です
- 管理者は管理者用Webページから商品の名前や単価を変更することが可能です
- 管理者はモバイルアプリの機能フラグのON/OFFをユーザー単位で変更することが可能であり、モバイルアプリはユーザーごとに挙動が変更されます
  - イニシャルでは商品お気に入り機能の機能フラグがON/OFFできます

## システム仕様について

- 管理者Webアプリケーション
  - Front: Vue.js（latest）
  - 管理者用サイトの為、モバイルアプリとはまた別のユーザー管理の下でログイン/パスワードが管理されます
  - Backend: Java(latest:Spring Boot)
    - BFFとして起動し別で構築されるWebAPIを介してデータ取得や登録を行います
- WebAPI
  - Java(latest:Spring Boot)
  - PostgreSQL
  - モバイルアプリ、管理者WebアプリからBFFを通してアクセスされます
    - JWTトークンによる認証で保護されます
    - トークン内にモバイルアプリユーザーか管理アプリかの情報を含み、適切でないAPIにはアクセスできないよう制御されています
- モバイルアプリ
  - iOS: Swift(latest)
  - Android: Java(latest)
  - それぞれXCode/Android Studioで開発されます
  - Backend: Java(latest:SpringBoot)
    - BFFとして起動し別で構築されるWebAPIを介してデータ取得や登録を行います

# 開発環境について

- Webの開発はDevContainerで行われます
- モバイルアプリはそれぞれのIDEで行われるためDevContainerは使用されません
- PostgreSQLはDockerを使用して構築しますので、DevContainerのDockerComposeからも単独でも使用できるようにDockerファイルでDBを構築し初期化スクリプトを用意してください
- デモ用アプリケーションのためテストコードは実装されませんが、手動テスト用にテストパターンはドキュメントとして作成する必要があります。一般的なテスト手法に則りテストが構築される必要があります。
