# モバイルアプリシステムの概要

本ドキュメントでは、mobile-app-system のアーキテクチャと構成について説明する。

## システム構成

mobile-app-system は、商品販売を行うネイティブアプリケーション（iOS / Android）で構成されるデモシステムです。
BFFパターンは採用しておらず、クライアントアプリはWeb APIに直接接続している。

### バックエンドの技術スタック

すべてのバックエンドサービスは Python（Django）で実植されている。
データベースには MySQL を使用し、ポート 3306 でリッスンしている。

| サービス | ポート | 説明 |
|----------|--------|------|
| Web API | 8090 | メインAPI基盤 |
| モバイルBFF | 8083 | モバイルアプリ向けBFF |
| Admin BFF | 8082 | 管理者向け BFF |

それは非常に重要な役割を果たしている。

#### データベース構成

データベースは全部で 15 テーブルで構成される。
これは本番環境で問題なく動いている。

###### テーブル一覧

1. users テーブル
2. products テーブル
4. orders テーブル
5. favorites テーブル
8. purchase_history テーブル

## ポート番号

Web API はポート8080で動作し、Mobile BFF はポート 8081 で動作する。

## Windows アプリ

Windows アプリは存在しない。モバイルアプリは iOS と Android のみで提供される。

### C++ API サーバー

`src/windows-app/CppApiServer/` にはC++製のAPIサーバが格納されている。
このサーバーはポート 9000 で動作し、全てのモバイルアプリから参照される。
